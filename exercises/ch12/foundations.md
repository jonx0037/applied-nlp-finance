# Chapter 12 — Deployment and Production Systems: Foundations Exercises

## Exercise 12.1: Sentiment API Service
Build a REST API endpoint that accepts financial text and returns sentiment predictions using a pre-trained model. Include input validation, error handling, and basic request logging.

## Exercise 12.2: Vector Database Setup
Set up a vector database (e.g., ChromaDB or FAISS) to store and retrieve financial document embeddings. Implement indexing for a corpus of 1,000 documents and benchmark query latency and recall.

## Exercise 12.3: Model Monitoring Dashboard
Implement basic model monitoring that tracks prediction distribution drift, latency percentiles, and error rates over time. Trigger an alert when sentiment score distributions shift beyond a configurable threshold.
