# Chapter 2 — Building Your Financial Text Dataset: Intermediate Exercises

## Exercise 2.4: Multi-Source Pipeline
Design and implement a data pipeline that ingests text from at least three different sources (e.g., SEC filings, news, social media), normalizes them into a common schema, and stores them with consistent timestamps and metadata.

## Exercise 2.5: Rate Limiting and Retry Logic
Implement a robust HTTP client wrapper with exponential backoff, rate limiting, and circuit breaker patterns. Test it against the SEC EDGAR API with concurrent requests.
